# Domain Docs

## Before exploring, read these

- **`CONTEXT.md`** at the repo root
- **`docs/adr/`** — 阅读即将工作的区域相关的 ADR

如果这些文件不存在，**继续执行**。不要标记缺失，不要建议预创建。

## File structure

Single-context:

```
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## Use the glossary's vocabulary

当输出命名领域概念时，使用 CONTEXT.md 中定义的术语。不要漂移到同义词。

## Flag ADR conflicts

如果输出与现有 ADR 矛盾，明确提出来而不是静默覆盖。
